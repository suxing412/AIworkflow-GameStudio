# 监制台（Producer Console）

**把软件生产流程交给 AI agent 流水线的驾驶舱。**
你只负责三件事——投池放行、争议拍板、品味终审；起草辅助、领单、执行、质检、
委托验收、失败兜底、记账、通知，全部自动。

[English](README.en.md) · [安装指南](套件/SETUP.md) · [设计与协议](docs/设计与协议.md)

```
你（制作人）          监制台（本工具）                agent（codex / claude 无头进程）
   起草/投池  ──────▶  工单池（目录即状态机）
                        │ 执行器每 15s 扫池
                        ├──▶ 空闲 agent 自动领单 ──▶ 按岗位协议执行，写回执
                        ├──▶ QA agent 只读复核（裁定通过/自修）
                        ├──▶ 委托单自动代核，通过直接完成
   拍板/终审  ◀──────  决策台（待定夺 / 保留单）
                        └──▶ 失败单进独立分诊通道，绝不悬空
```

## ⚠️ 使用前必读（安全披露）

1. **实弹模式下 agent 对你注册的项目仓库有完全写权限**：执行器以
   `codex exec --dangerously-bypass-approvals-and-sandbox` / `claude -p --permission-mode acceptEdits`
   启动无头 CLI 并把工作目录指向你的项目仓库。请只注册你愿意让 agent 写入的仓库，
   并保持仓库有 git 历史可回滚。实弹默认**锁定**，解锁是显式操作（参数页开关，二次确认）。
2. **本工具会读写你的 CLI 凭据文件**：为显示订阅额度，会读取
   `~/.claude/.credentials.json` 并在 token 过期时自动续期（与官方 CLI 同一 OAuth 流程，
   续期结果写回该文件）；会读取 `~/.codex/config.toml` 获取模型配置。凭据永不上传、不离开本机。
3. **自动记账**：若安装目录在 git 仓库内，工单流转记录每 10 分钟自动 `git commit`
   （只 commit 不 push，可在配置关闭）。
4. 试跑模式（默认）零 AI 调用、零额度消耗，可安全体验全流程。

## 快速开始

1. 从 [Releases](../../releases) 下载 `监制台-套件-vX.Y.Z.zip` 并解压
2. 双击 **部署.bat**：选安装目录 →（可选）注册第一个项目仓库 → 自动启动
3. **验收标准只有一条：总览右上角「环境」显示 就绪**（背后是四组全链路自检，
   降级/阻断时逐项写明修复动作）

前置要求（详见 [SETUP.md](套件/SETUP.md)）：Windows 10/11、codex CLI、
claude CLI（Claude Code）已登录、代理（如你的网络需要，自动解析注入）。

## 配置详情（studio.config.json 全字段）

所有配置均可在应用内 **参数页（右上角 ⚙）** 调整，明文 JSON 是唯一事实源。

| 字段 | 默认 | 说明 | UI 位置 |
|---|---|---|---|
| `server.port` | 4270 | 本地服务端口（仅监听 127.0.0.1） | 参数页·服务端口（重启生效） |
| `职能` | 策划/程序/美术/QA/装配 | 职能全集；职能由**产出写区**决定 | —（协议级，改配置） |
| `执行池.codex.职能` | [程序] | 走 codex CLI 的职能 | —（协议级） |
| `执行池.claude.职能` | [策划,美术,QA,装配] | 走 claude CLI 的职能 | —（协议级） |
| `执行池.*.阈值` | codex 70 / claude 50 | 5h 用量 ≥N% 自动冻结该池领单（额度锁） | 参数页·执行池阈值 |
| `执行池.*.周阈值` | 90 | 周用量阀门，烧穿周额度是灾难级 | 参数页·执行池阈值 |
| `闸值.待验收积压闸` | 8 | 待验收 ≥N 停止建议投放 | 参数页·参数闸值 |
| `闸值.QA自修上限` | 2 | QA 不过自修轮数，超则上交人裁 | 参数页·参数闸值 |
| `闸值.滞留超时小时` | 4 | 在途单超时告警（只告警不撤回） | 参数页·参数闸值 |
| `推荐.精力档` | 高 | 低=推荐固定 1 张专注；高=随你处理速度爬档 | 参数页·推荐在途 |
| `推荐.速度窗口小时` / `每档处理数` | 2 / 2 | 处理速度统计窗口与爬档步长 | 参数页·推荐在途 |
| `网络.代理默认` | http://127.0.0.1:7890 | 代理解析链的兜底（环境变量→注册表→此值） | 改配置 |
| `执行器.间隔秒` | 15 | 扫池周期（领单+起执行） | 参数页·执行器 |
| `执行器.实弹解锁` | **false** | 权力开关：true 才允许切实弹烧额度 | 参数页·实弹解锁（二次确认） |
| `执行器.执行超时分钟` | 30 | 实弹单超时树杀 → 进执行失败态 | 参数页·执行器 |
| `执行器.记账间隔分钟` | 10 | 自动 git commit 周期（0=关） | 参数页·执行器 |
| `执行器.试跑耗时秒*` | 3–8 | 试跑模拟执行的耗时区间 | 改配置 |
| `quota.claudeMinIntervalSeconds` | 300 | 额度查询最小间隔（防限流硬保证，下限 120） | 参数页·额度刷新间隔 |
| `模型.claude默认` / `codex默认` | sonnet / 空 | 执行体力档（空=CLI 自带默认） | 参数页·模型档 |
| `模型.质检` / `代核` | opus | 裁判档（判断活用强模型） | 参数页·模型档 |
| `模型.可选.*` | — | 模型下拉的候选清单（自动监测+手动增补） | 参数页·可选模型增补 |
| `项目.注册` | {} | 项目名 → 仓库路径（agent 的写入目标） | 参数页·项目注册 |
| `项目.默认` | — | 起草时的默认项目 | 参数页·项目注册 |
| `agents[]` | 每职能 1 人 | 编制即并发上限；`模型` 字段可个体覆盖 | 参数页·职能编制/编制表 |

## 目录结构（安装后）

```
AI工作室/
  监制台 X.Y.Z.exe     ← 主程序（绿色，双击即用）
  studio.config.json   ← 全部配置（明文 JSON）
  草稿/ 待投/ 池/ 在途/ 质检/ 待验收/ 待定夺/ 执行失败/ 完成/ 已归档/
                       ← 目录即状态机：工单 = 状态文件夹里的 .md
  回执/                ← 每单的完工报告
  岗位协议/            ← 各职能 agent 章程（md 明文，改了下一单生效）
  风格库/              ← 精选标杆与美术范本（验收后人工入库）
  journal/             ← 只追加的操作流水
```

## 核心概念

- **目录即状态机**：工单状态 = 所在文件夹，改状态 = 原子改名；明文 .md 是唯一事实源
- **拉取制**：agent 干完自动领下一张；编制人数 = 并发上限
- **双闸**：暂停闸门（手动）+ 额度锁（用量超阈自动冻结领单）
- **模型分级**：便宜模型干体力（执行），贵模型当裁判（QA 复核/委托代核）
- **执行失败态**：CLI 崩溃/超时的单进独立分诊通道（重投/上呈/废弃），绝不悬空
- **精选制风格库**：验收完成的精华由人工提炼入库，agent 领单前对齐

完整协议见 [docs/设计与协议.md](docs/设计与协议.md)。

## 已知限制

- 仅 Windows（依赖 Windows 路径/进程语义；exe 为 win-x64 便携版）
- 订阅额度端点有账号级限流：读数按 ≥5 分钟纪律刷新并标注时间戳，偶有陈旧属正常
- 无自动更新：新版本重跑 部署.bat 覆盖 exe 即可（配置与工单不受影响）
- 美术职能暂无图像生成接入（产出规格文档+占位资产）

## 卸载

删除安装目录即卸载（所有数据都在里面）。本工具不写注册表、不装服务。

