# 设计与协议（蒸馏版）

监制台的完整协议在实践中经四十条逐条决议演化而成，本文是给使用者的蒸馏版：
理解这些概念，就理解了这个工具的全部行为。

## 一切皆工单

生产的最小单位是**工单**：一个带 frontmatter 的 .md 文件。必填字段：职能、优先级、
QA 开关、验收方式、项目；正文四段：范围 / 不要做 / 验收标准 / 完工要求。
明文即事实源——任何时刻你都可以用文本编辑器直接检查、修正系统状态。

## 目录即状态机

工单的状态由它所在的文件夹决定，共十态：

```
草稿 → 待投 → 池 → 在途 → 质检 → 待验收 → 完成
                 ↓      ↘ 执行失败 ↗  ↓
               （收回）    （分诊）   待定夺 → 已归档
```

改状态 = 原子改名（rename）。并发领单竞态由文件系统的原子性天然兜底：
两个 agent 同抢一张单，第二个必然失败。没有数据库，没有第二事实源。

## 拉取制（而非派发制）

agent 不等人派活：执行器周期扫池，空闲 agent 自动领取本职能队首可领单，
干完交产出，回来继续领（自动续单）。制作人只把控两个闸：
**投池**（放行哪些单进池）与**闸门**（何时停）。

- **一人一张**：每个 agent 同时只持一张单（含质检/待定夺中的）
- **编制即上限**：并发度 = 在岗 agent 数，参数页直接加减人
- **依赖闸**：依赖单未完成的工单在池里领不走

## 职能与写区

**职能由产出写区决定，不由技能类型决定**——这是边界不糊的关键：

| 职能 | 产出 | 写区 |
|---|---|---|
| 策划 | 设计文档 | 文档区 |
| 程序 | 脚本 + 测试 | 代码区（不碰拼接面） |
| 美术 | 资产/规格 | 资产区 |
| 装配 | 能跑的组装体 | 拼接面（场景/预制体/工程设置/数据接入）**独占** |
| QA | 核验结论 | 只读 |

拼接面独占是并发隔离的结构性保证：高冲突文件只有一个属主。
生产线自然成链：策划 → 程序+美术（并行）→ 装配 → QA → 验收。

各职能的行为章程在 `岗位协议/`（明文 md，执行器组提示词时自动前置，改了下一单生效）。

## 双闸与额度纪律

- **暂停闸门**：手动，一键冻结领单（在途单继续跑完）
- **额度锁**：自动，某 CLI 池 5h/周用量超阈值即冻结该池领单，重置后自动解冻
- 订阅额度查询遵守 ≥5 分钟硬间隔 + 失败指数退避 + 跨重启持久化——
  官方端点有账号级限流，读数偶尔陈旧（带时间戳标注）是纪律的代价，不是故障

## 质量与验收

- **QA 是单内可选关卡**：QA=开 的单交产出后进质检，QA agent 只读复核；
  不过则原单回在途自修（≤2 轮），修不好上交待定夺给人裁
- **验收委托制**：验收方式=委托 的单由 AI 代核（按验收标准逐条核验，通过自动完成，
  不过留给人裁）；=保留 的单只有制作人能签——品味永远不外包
- **返工不复活**：验收打回的单归档，另开新单带回链；旧单永不复活

## 失败与兜底

实弹执行中 CLI 崩溃/超时（默认 30 分钟树杀）的单进**执行失败**态：
纯本地目录改名（断网也能落位），带失败原因/次数，等待分诊——
重投（清主办回池）/ 上呈（给人拍板）/ 废弃。任何单都不会悬空。

## 追溯链

四条链把一切串起来：**父子**（母单-子单）、**返工自**（新单回链旧单）、
**依据**（工单 → 设计文档#锚号）、**依赖**（单间先后）。
上游设计改版时可按锚号批量标记下游工单「待复核」——
被标记的单领不走、交不了，直到人工确认核对过新版。

## 模型分级

便宜模型干体力（执行，默认 sonnet），贵模型当裁判（QA 复核/委托代核，默认 opus）。
三级解析：agent 个体覆盖 > 池默认 > CLI 自带默认。裁判档不接受个体覆盖。

## 精选制风格库

验收完成的精华由制作人**手工提炼**入库（入库是审批动作）：
策划单 → 标杆条目（一句话公理，带来源单号）；美术/装配单 → 范本文件。
agent 领单前先读风格库对齐。只进精品，可进可出。

## 审批点全集

全系统只有五个人工审批点，其余转移全自动：
①投池 ②待定夺拍板 ③保留单终审 ④入库/入标杆 ⑤撤回/废弃。
