# AI 工作室（监制台）安装指南

一个把游戏开发流程交给 AI agent 流水线的驾驶舱：你只负责三件事——**投池放行、待定夺拍板、保留单终审**；
起草辅助、领单、执行、质检、委托验收、失败兜底、记账、通知全部自动。

## 前置要求

| 项 | 说明 |
|---|---|
| Windows 10/11 | 监制台是绿色 exe，免安装 |
| codex CLI | 程序职能的执行引擎。安装后 `codex --version` 能出版本号即可 |
| claude CLI（Claude Code） | 策划/美术/QA/装配职能 + 质检/代核引擎。装在 `~/.local/bin/` 或 PATH 均可（监制台按绝对路径解析，不依赖 PATH） |
| claude 登录 | `"%USERPROFILE%\.local\bin\claude.exe" auth login`（若终端无代理先 `set HTTPS_PROXY=http://127.0.0.1:7890`） |
| 代理（如在需要的网络环境） | 监制台启动时按 环境变量 → 系统注册表 → 配置默认（127.0.0.1:7890）自动解析注入，通常无需手动设置 |

## 安装

1. 解压本压缩包到任意位置
2. 双击 **部署.bat**，按提示：选安装目录 → （可选）注册第一个项目仓库
3. 监制台自动启动

**验收标准只有一条：总览右上角「环境」显示 就绪。**
它背后是四组全链路自检（运行时与 CLI / 凭据与额度 / 项目与目录 / 协议与配置）——
全绿即整个 app 可用。降级/阻断时悬停看原因，或进 设置（右上角 ⚙）看逐项详情，每项都写了修复动作。

## 多项目：一份监制台管所有项目

**新项目不用重装。** 在参数页「项目注册」里加第二个项目仓库后，监制台启动会先落**项目启动页**：
每个项目一张监控卡（在途/待验收/待定夺计数 + 需处理红点），点卡进入该项目的驾驶舱，
所有视图只显示该项目的工单；点左上角 logo 随时回启动页换项目。
执行器、agent 编制、额度双闸是账号级共享资源，全局只有一套（全局配置入口 = 启动页右上角 ⚙）。
只注册一个项目时跳过启动页直接进驾驶舱。

## 首次使用（5 分钟上手）

1. **参数页（⚙）**：确认项目注册、职能编制（默认每职能 1 人）、执行器在运行（绿呼吸灯）
2. **工单池**：点「＋ 起草」开第一张单（职能/QA 开关/验收方式/项目），存为待投 → 投池
3. 执行器 15 秒内自动领单。**默认是试跑模式（零额度）**——全流程真实流转但不真调 AI
4. 走顺后再开实弹：参数页「实弹解锁」→ 执行模式切「实弹」——此后 agent 真调 codex/claude 干活烧额度
5. 你在 **决策台** 验收（委托单会被自动代核，通过直接完成）；产出精华在完成单详情页「入标杆 / 入美术库」

## 核心概念速览

- **目录即状态机**：工单 = 安装目录下九个状态文件夹里的 .md 文件，改状态 = 挪文件夹；明文即事实源，随时可手工检查
- **拉取制**：agent 干完自动领下一张（编制人数 = 并发上限，参数页直接调）
- **双闸**：暂停闸门（手动）+ 额度锁（codex/claude 用量超阈值自动冻结领单）
- **执行失败态**：CLI 崩溃/超时的单进独立通道等分诊（重投/上呈/废弃），不会悬空
- **岗位协议**：`岗位协议/` 下的 md 是各职能 agent 的章程，改了下一单立即生效
- **自动记账**：若安装目录在 git 仓库内，工单流转每 10 分钟自动 commit（只 commit 不 push）

## 常见问题

- **环境显示降级：代理未解析** → 确认代理软件在跑，或把配置里 `网络.代理默认` 改成你的代理地址后重启
- **claude 凭据红灯** → 终端跑 `claude auth login`（见前置要求第 4 行）
- **额度读数陈旧** → 正常现象：官方额度端点有账号级限流，监制台按 ≥5 分钟纪律查询并显示最近读数+时间戳，等下个窗口自动恢复
- **端口被占** → 配置里 `server.port` 改一个（参数页也能改），重启生效
- **想换机器/给同事** → 整个安装目录拷走即可（工单/回执/配置全在里面）；或重新跑一遍本套件
